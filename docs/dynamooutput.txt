(Ben notes):
This is a 1KB item from DynamoDB. If it gets any bigger, it will consume 2 WRUs when modified.

258 bytes for keys 'cmpnt', 'gamId', 'name', 'maxSz', 'isOpn' and their vals

230 bytes for key 'text' and the text block after it

8 bytes for the key 'sesns' (5 bytes) and overhead from pointing to a map (3 bytes)

112 bytes for the 'sesns' map keys (8 keys * (10 chars + 1 overhead per key + 3 from themselves being maps))

56 bytes for the 'sesns' submap keys (8 keys *(5 characters + 2*(1 overhead per submap key)))

360 bytes for the 'sesns' submap vals (8 keys *(36 + 9 characters))

= 1024 bytes


(Command, after manually setting up the table and item):
aws dynamodb update-item \
  --table-name "testgames" \
  --key '{
    "gamId": {"S": "grouping_flankers_swindler"},
    "cmpnt": {"S": "LOBBY"}
  }' \
  --update-expression "SET maxSz = :val" \
  --expression-attribute-values '{
    ":val": {"N": "6"}
  }' \
  --return-consumed-capacity TOTAL --return-values ALL_NEW \
  --profile {awsProfileName}
